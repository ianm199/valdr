What it is

valkey-rs is a from-scratch reimplementation of the Redis/Valkey server in Rust — RESP2/3 protocol, all core data types, streams, pub/sub, transactions, Lua scripting, ACL, eviction, TLS, RDB + AOF persistence, and replication basics. It runs as an ordinary redis-server binary that existing Redis clients talk to unchanged. ~80k lines of Rust stand in for upstream's ~187k lines of C.

Thinking about adopting? The 877 above is the pass rate within the slice we've built, not coverage of all upstream Valkey behavior. There is no clustering, no loadable C modules, and replication isn't gated yet. Read Scope and gaps before wiring it into anything load-bearing.

Run it

# from the published container image
$ docker run --rm -p 6379:6379 -v valkey-rs-data:/data ghcr.io/ianm199/valkey-rs:alpha

# or build from source
$ cargo build --release
$ ./target/release/redis-server --port 6379 --bind 127.0.0.1

# then use any ordinary Redis client
$ redis-cli set hello world
OK
$ redis-cli get hello
"world"

Conformance

The load-bearing claim. Three independent oracles re-verify the port against the real upstream binary on every commit — build success is never the signal.

RESP wire-diff

23 protocol corpus scripts are sent to both the C Valkey server and valkey-rs on parallel sockets; the raw RESP replies are compared byte-for-byte. Any single-byte divergence fails.

RDB bidirectional

We save → C loads, and C saves → we load, across every encoding (listpack, intset, skiplist, quicklist, streams). The resulting keyspaces are diffed.

Safety model

A safe public surface over a tiny, audited unsafe core. Four of the six crates compile with zero unsafe. The only unsafe blocks — 5 total — wrap Unix fork(2) and waitpid(2) for background save / rewrite flows, each with a // SAFETY: invariant and a per-crate budget enforced by a hook.

How it was built

The server is the artifact; the AI-agent porting harness is the method. Bounded single-purpose agents make small commits, gated by mechanical safety hooks and — non-negotiably — by compatibility oracles. The important claim is not that AI wrote the code; it is that every change is continuously checked against real upstream Valkey behavior.

Oracle-gated, not vibes-gated

A change counts only when a wire-diff, RDB, or TCL oracle says it matches the reference C binary. Survey runners record abort / no-summary cases separately so no number hides behind an aggregate.

Mechanical guardrails

An unsafe-budget ceiling, forbidden-pattern bans, and required status trailers are enforced as hooks that fail the tool call on violation.
